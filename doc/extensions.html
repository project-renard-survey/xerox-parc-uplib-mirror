UpLib Extensions Help

 Searching · Collections · Extensions · FAQ UpLib 1.7.11  · PARC / ISL

The UpLib system can be changed by adding modules of code called extensions to repositories. An extension can modify existing functionality or add new functionality. Extensions are written in the Python programming language. There are two basic classes of extensions: initializers, which change or augment the underlying automatic functionality of the repository, and actions, which add functions to be explicitly invoked by a client of the repository service.

Invoking an extension.
All actions have URLs associated with them, of the form /action/module-name/function-name. The module-name is the name of the Python module containing the function; the function-name is the name of the function being invoked. You can invoke an action simply by typing the URL into your browser. For example, if your repository was at port 9090 of host abc.com, you could invoke the action foo.bar by visiting the URL https://abc.com:9090/action/foo/bar.

You can pass parameters to an action function as query parameters to the URL. For instance, suppose you had a function print in module utils. You'd like to invoke it with two parameters, the document ID of the document to print (doc-id), and the name of the printer you want the document printed on (printer). So you'd visit the URL https://abc.com:9090/action/utils/print?doc-id=01065-57-5728-074&printer=laser1.

It's often clumsy to type in parameters such as document ID. You can avoid this by adding buttons which encapsulate action function calls to the small window in the upper left-hand side of the page view of a document. See the document called USAGE for information on the configuration option "default-html-controls-template-file".

Viewing active extensions.
If you go to the bottom of an UpLib repository view, you'll find a button there called "Manage Extensions". Pressing it will take you to the Extensions Manager, which shows you all the extensions currently active in your repository. Extensions can come from three places. The first is a site-wide extensions site, which is used by site administrators for extensions they feel everyone should have. The second is repository-specific; extensions installed in a particular repository can be either active or inactive. Finally, each repository can have a list of directories, called its extensions-path. A panel at the bottom of the Extensions Manager allows the user to add to or remove from this list. Any extension in any of these directories will also be active in that repository.

The Extensions Library.
It's possible to set up a library of extensions which users can contribute extensions to, or download extensions from. Pressing the button at the top of the Extensions Manager labelled "Find New Extensions to Install" will take you to a view of the extensions available in the library.

But be careful! Extensions are very powerful, and there's no virus scanning in UpLib. Only download extensions from trusted authors!
